# SPC-E-water-model-CHARMM-NAMD
Procedure for Solvating a System with the SPC/E Water Model in NAMD and Performing Molecular Dynamics Simulations Using the CHARMM Force Field
# Preparation of an SPC/E Water Box for NAMD (CHARMM Force Field)

## Overview

This repository documents the complete workflow used to prepare an **SPC/E-solvated biomolecular system** for molecular dynamics simulations in **NAMD** while using the **CHARMM force field**.

Since CHARMM does not natively provide an SPC/E topology for NAMD, a custom SPC/E topology and parameter set was generated by translating the original SPC/E model into CHARMM format.

```text

Generate CHARMM Protein Topology
        │

        │
        ▼
Convert GRO → PDB
        │
        ▼

        │
        ▼

        │

        │

        │

        │
```

## Step 1: Solvate the Protein in GROMACS

Generate the simulation box:

```bash
gmx editconf \
  -f protein.gro \
  -o protein_box.gro \
  -c \
  -d 1.2 \
  -bt cubic
```

Solvate using SPC/E water:

```bash
gmx solvate \
  -cp protein_box.gro \
  -cs spc216.gro \
  -o system_spce.gro \
  -p topol.top
```

### Why GROMACS?

GROMACS provides a fast and reliable solvent packing algorithm.

Although the solvent coordinate file is named `spc216.gro`, the actual water model is determined by the selected force field parameters.

**Output:**

```
system_spce.gro
topol.top
```

---

## Step 2: Convert GRO to PDB

Convert coordinates into PDB format for VMD/psfgen:

```bash
gmx editconf \
  -f system_spce.gro \
  -o system_spce.pdb
```

**Output:**

```
system_spce.pdb
```

---

## Step 3: Rename Water Molecules

GROMACS water names:

```
SOL
OW
HW1
HW2
```

were converted into:

```
SPCE
OH2
H1
H2
```

using VMD.

### Why?

`psfgen` requires that residue names and atom names match those defined in the topology file.

---

## Step 4: Separate the System

The complete system was separated into:

```
protein_only.pdb
water_only.pdb
sodium_only.pdb
chloride_only.pdb
```

### Why?

Each component is generated as an independent PSF segment.

---

## Step 5: Split Water into Multiple Segments

The solvated system contained:

```
22561 water molecules
```

A standard PDB file stores residue IDs using only four columns.

Maximum residue number:

```
9999
```

Therefore residue numbering wraps after 9999, producing a `duplicate residue key` error in psfgen.

To solve this, waters were divided into:

```
water_W01.pdb
water_W02.pdb
water_W03.pdb
```

Each file contains fewer than 9000 waters. Residue numbering restarts from 1 for every segment.

---

## Step 6: Create the SPC/E Topology

A custom topology file, `top_spce.rtf`, was written.

The topology defines:

- residue name
- atom names
- atom types
- charges
- connectivity

**Example:**

```
RESI SPCE

ATOM OH2
ATOM H1
ATOM H2
```

---

## Step 7: Create the SPC/E Parameter File

A custom parameter file, `par_spce.prm`, was generated.

The parameter file contains:

- bond parameters
- angle parameters
- Lennard-Jones parameters

The physical values were taken from the original SPC/E model and converted into CHARMM format where necessary (e.g., σ → Rmin/2 and kJ/mol → kcal/mol).

---

## Step 8: Generate the Water PSF

Load the topology:

```tcl
topology /path/to/top_spce.rtf
```

Generate water segments:

```
W01
W02
W03
```

using psfgen.

**Output:**

```
water_spce.psf
water_spce.pdb
```

---

## Step 9: Generate the Complete System

The final system was generated by combining:

```
Protein + SPC/E Water + Na+ + Cl−
```

**Output:**

```
system.psf
system.pdb
```

---

## Step 10: Define Periodic Boundary Conditions

The PSF and PDB files do **not** store the periodic simulation box automatically.

The box vectors were obtained from the last line of the original GROMACS `.gro` file.

**Example:**

```
10.2456 9.8762 12.8000
```

GROMACS stores dimensions in **nm**. Convert to **Å**:

```
102.456
98.762
128.000
```

Use these values in NAMD:

```tcl
cellBasisVector1   102.456   0.0      0.0
cellBasisVector2     0.0    98.762    0.0
cellBasisVector3     0.0     0.0    128.000

cellOrigin          Xc       Yc       Zc
```

## Important Notes

### Water Segments

Splitting water into `W01`, `W02`, `W03` is only required because of the PDB residue-number limitation.

It has **no effect** on molecular dynamics or periodic boundary conditions.

### Why Custom SPC/E Files?

CHARMM provides a topology for its modified TIP3P water model. SPC/E is a different water model.

Therefore, dedicated `top_spce.rtf` and `par_spce.prm` files were created by translating the published SPC/E model into CHARMM topology and parameter syntax.

## Common Errors

| Error | Cause | Solution |
|---|---|---|
| **Duplicate residue key** | More than 9999 residues in one PDB segment. | Split waters into multiple segments. |
| **Unknown residue type SPCE** | `top_spce.rtf` was not loaded. | Provide the correct absolute path to the topology file. |
| **Unknown atom** | PDB atom names do not match topology. | Rename `OW`→`OH2`, `HW1`→`H1`, `HW2`→`H2`. |
| **Suspicious PBC side length** | PDB does not contain periodic box information. | Specify `cellBasisVector1`, `cellBasisVector2`, `cellBasisVector3`, and `cellOrigin` in the NAMD configuration. |
